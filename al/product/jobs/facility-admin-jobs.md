# Jobs to Be Done: Facility Admin

Format: "When [situation], I want to [motivation], so I can [expected outcome]."

---

**Job 1 — User onboarding**
When a new Approving Officer or Gate Security guard joins the facility, I want to create their account, assign their role and department scope, and have them operational in under 5 minutes, so gate operations are never blocked waiting for IT setup.

---

**Job 2 — Compliance audit export**
When a compliance officer or auditor requests a record of all visitor activity for a specific date range or gate, I want to export a complete, immutable audit log as a CSV or PDF without any manual data compilation, so I can respond to the request the same day without involving other staff.

---

**Job 3 — Pass rule configuration**
When the facility's access policy changes — new valid window defaults, different advance booking periods, or updated acceptable ID types — I want to update the rules in one configuration screen and have them apply immediately to all new requests, so there is no gap between policy and system behaviour.

---

**Job 4 — Incident alerting**
When an anomalous event occurs — a repeated failed scan at a gate, a MANUAL\_ENTRY spike, or a system outage lasting more than 30 minutes — I want to receive an alert with enough context (gate, time, count, type) to assess severity immediately, so I can respond before the situation escalates to a formal incident.

---

**Job 5 — Data retention oversight**
When the retention period for visitor records is about to trigger automatic deletion, I want to receive a notification 48 hours in advance listing what will be deleted, so I can confirm the deletion is correct or pause it if a record is needed for an ongoing investigation.
